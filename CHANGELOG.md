# История версий

Формат вдохновлён [Keep a Changelog](https://keepachangelog.com/ru/);
версии — [SemVer](https://semver.org/lang/ru/). Полные диффы — в
[релизах GitHub](https://github.com/xvin84/Adalight/releases).

## [0.23.0] — 2026-07-21

- Исправлено: на Windows встроенные моды (эффекты лампы, цветомузыка, захват
  экрана, транспорты) не находились — приложение сообщало «не удалось найти
  плагин» и по сути не работало. Причина — динамическая загрузка модов по
  строковому имени, которую сборщик `.exe` (PyInstaller) не видел и не
  упаковывал в дистрибутив. Теперь моды подключаются статическими импортами и
  всегда попадают в сборку. На Linux ошибка не проявлялась (запуск из
  исходников).

## [0.22.0] — 2026-07-21

- Умный выбор порта: по умолчанию в списке показаны только платы (Arduino/ESP)
  и USB-устройства — с понятными ярлыками («COM5 — Arduino Uno»), а мусорные
  системные порты (ttyS*) скрыты. Распознавание по USB-дескриптору (VID/PID),
  как «Get Board Info» в Arduino IDE — плату не трогаем и не открываем.
- Галочка «Показать все порты» — если платы нет в списке (экзотический чип).
- Пометка бета-версии в заголовке окна и в окне «О программе».
- В репозиторий добавлена прошивка (firmware/) — эталонный скетч Adalight
  авторства AlexGyver (alexgyver.ru/arduino_ambilight), с указанием авторства.
- Тело GitHub-релиза теперь подробное: «что нового», описание и возможности,
  сначала на английском, затем на русском.

## [0.21.0] — 2026-07-19

- «Всё есть мод», этап 5 (завершение миграции): событийная шина. Моды и плагины
  могут реагировать на состояние приложения и друг на друга, не завися от
  внутренностей.
- Плагин может подписаться на события (`api.on`) и рассылать свои (`api.emit`):
  запуск/остановка подсветки, кадр, уведомление, доступное обновление.
- Пример плагина на событиях — «Лог уведомлений» в `examples/plugins`.
- Починка CI: в раннер доставляются системные библиотеки Qt (тесты, трогающие
  интерфейс, снова выполняются на CI).

## [0.20.0] — 2026-07-19

- «Всё есть мод», этап 4: способы доставки цветов на ленту (serial по USB и
  WLED по Wi-Fi) стали встроенным модом «Транспорты» — виден и управляем в
  менеджере плагинов.
- Плагин может добавить свой транспорт (`register_transport`).
- Ядро (`device.py`) стало тонким фасадом: конвейер цвета остаётся в ядре, а
  доставку байт берёт транспорт из реестра; выбор транспорта в интерфейсе
  строится из реестра, поля подключения показываются по типу транспорта.

## [0.19.0] — 2026-07-19

- «Всё есть мод», этап 3: источники захвата экрана стали встроенным модом
  «Захват экрана» — виден и управляем в менеджере плагинов.
- Плагин может добавить свой источник захвата.
- Headless-режим (CLI) теперь тоже поднимает встроенные моды — эффекты и
  захват работают и без графического интерфейса.

## [0.18.0] — 2026-07-19

- «Всё есть мод», этап 2: эффекты режима «Цветомузыка» стали встроенным модом
  «Цветомузыка» — виден и управляем в менеджере плагинов.
- Плагины могут добавлять свои эффекты цветомузыки тем же способом, что и
  встроенные.

## [0.17.0] — 2026-07-19

- Режим сна: на статичной картинке лента больше не гаснет — приложение
  поддерживает плату «в тонусе» (keep-alive). Раньше через ~10 секунд без
  изменений плата сама выключала ленту.
- Если режим сна включить, лента, наоборот, гаснет при простое — время
  задаётся слайдером («Яркость» → «Режим сна»).

## [0.16.0] — 2026-07-19

- «Всё есть мод», этап 1: эффекты режима «Лампа» стали встроенным модом
  «Эффекты лампы» — он виден и управляем в менеджере плагинов.
- Моды включаются и выключаются; выключение реально убирает возможности
  (с предупреждением при выключении базовых модов).
- Плагины добавляют свои эффекты тем же способом, что и встроенные.

## [0.15.0] — 2026-07-19

- Плагины могут добавлять свои эффекты лампы — они появляются в списке эффектов
  наравне со встроенными («всё есть мод»); пример — examples/plugins/plasma_effect.py.
- Встроенные эффекты лампы переведены на общий реестр эффектов.
- Окно «Что нового» при обновлении с версии до его появления показывает всю
  историю до текущей, а не только текущую версию.

## [0.14.0] — 2026-07-19

- Окно «Что нового» после обновления: список изменений (в т.ч. через несколько
  версий) на языке интерфейса, с переключателем языка прямо в окне.
- Плагины из каталога ставятся без перезапуска; установленные показываются
  серыми с пометкой «Установлено».
- Языки видны в списке «Установленные» менеджера — «Сделать языком интерфейса»;
  английский идёт через тот же контракт `create_locale()`, что и плагины-локали.
- Языки сообщества ставятся из каталога и появляются в списке сразу.
- Исправлена вёрстка настроек уведомлений (обрезанная подсказка, наложение
  пикера позиции при широком шрифте).

## [0.13.0] — 2026-07-19

- Локализация интерфейса: русский и английский (Система → Язык), смена языка
  применяется после перезапуска.
- Языки — это плагины: свой перевод добавляется файлом с `create_locale()`
  в папке плагинов; шаблон — examples/locales/en.py.
- Английский встроен и доступен сразу, без установки.

## [0.12.0] — 2026-07-18

- Менеджер плагинов отдельным окном: «Установленные» (вкл/выкл, настройки,
  удаление) и «Каталог» (поиск, установка). Вкладка в сайдбаре — сводка.
- Плагины объявляют `settings_schema` — менеджер строит форму настроек сам,
  без кода GUI (первый шаг к «всё есть плагин»).
- Вспышка уведомлений «бульк»: капля с расходящейся по ленте волной (стиль
  на выбор — «бульк» или «пятно»).
- Позиция вспышки задаётся перетаскиванием по краю экрана (только периметр).
- Подсказка в настройках: ловятся только системные уведомления Windows.
- Кнопки «Сообщить об ошибке» / «Предложить идею» в «Системе» открывают
  преднаполненный GitHub-issue с диагностикой.

## [0.11.0] — 2026-07-17

- Позиция вспышки уведомлений задаётся перетаскиванием пятна по схеме экрана
  (с мгновенной пробной вспышкой на ленте при отпускании).
- Каталог плагинов: официальные и от сообщества, установка в один клик.
- Режим «любые приложения»: вспышка цветом иконки приславшего уведомление.
