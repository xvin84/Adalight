# Adalight

[English](README.md) | **Русский**

Фоновая подсветка (ambilight) для LED-ленты за монитором: приложение
захватывает экран, усредняет цвета по краям и отправляет их на Arduino/ESP
по классическому serial-протоколу **Adalight**.

Работает на **Windows** (DXGI Desktop Duplication через `dxcam`, fallback —
`mss`) и **Linux/Wayland** (Hyprland/wlroots: `wf-recorder`, fallback —
`grim`). GUI на Qt (PySide6): все настройки в одном окне, живой предпросмотр
раскладки диодов и иконка в трее — подсветка работает и с закрытым окном.

## Возможности

- Serial-порт с автообнаружением, скорость, **порядок каналов** (RGB/GRB/BGR/…) —
  ленты WS2812 обычно ожидают GRB.
- Количество диодов на каждой стороне, стартовый угол, направление ленты
  (по/против часовой), зеркалирование по X/Y.
- Выбор монитора, целевой FPS, гамма / яркость / насыщенность / сглаживание.
- Тестовые режимы для калибровки: заливка сторон цветами и «бегущий диод».
- Headless-CLI для автозагрузки; GUI и CLI используют общий JSON-конфиг
  (`%APPDATA%\adalight\config.json` на Windows, `~/.config/adalight/` на Linux).

## Скачать (Windows)

Возьмите `Adalight.exe` из [последнего релиза](https://github.com/xvin84/Adalight/releases) —
Python не нужен.

## Запуск из исходников

Нужен [uv](https://docs.astral.sh/uv/):

```bash
uv sync

# GUI
uv run main.py

# headless и сервисные режимы
uv run main.py --live
uv run main.py --sides          # тест: верх=красный, право=зелёный, низ=синий, лево=жёлтый
uv run main.py --chase          # тест: бегущий диод
uv run main.py --off            # погасить ленту
uv run main.py --list-monitors
uv run main.py --list-ports
```

## Калибровка

1. Запустите **«Тест: стороны»**: верхний край должен гореть красным, правый —
   зелёным, нижний — синим, левый — жёлтым. Если стороны перепутаны, поменяйте
   стартовый угол, направление или зеркалирование — предпросмотр в окне
   отражает изменения сразу.
2. Если перепутаны *оттенки* (красный горит зелёным и т.п.), поменяйте порядок
   каналов — для WS2812 обычно GRB.
3. **«Тест: бегущий»** гоняет одну яркую точку по ленте — так проверяется
   точный порядок диодов; первый диод помечен кольцом в предпросмотре.

## Как это работает

- Заголовок `"Ada" + count_hi + count_lo + (hi^lo^0x55)`, дальше по 3 байта на
  диод — стандартный протокол Adalight, совместим с популярными скетчами
  для Arduino.
- Краевые зоны предвычисляются под раскладку; каждый кадр усредняется по
  зонам, сглаживается экспоненциально и корректируется по гамме через
  предвычисленный LUT (без `pow` на каждый кадр).
- Цикл захвата/обработки не зависит от Qt (`adalight/engine.py`): GUI гоняет
  его в фоновом потоке, CLI — напрямую.

```
adalight/
  config.py        # dataclass конфигурации + JSON load/save
  geometry.py      # раскладка диодов и зоны захвата
  device.py        # протокол Adalight, LUT-гамма, порядок каналов
  engine.py        # цикл захват -> обработка -> отправка (без Qt)
  capture/         # бэкенды: dxcam (Windows), mss, wf-recorder, grim
  cli.py           # headless-режимы
  gui/             # PySide6: главное окно, предпросмотр, трей
main.py            # входная точка: GUI без аргументов, иначе CLI
```

## Релизы

- **Release** (`.github/workflows/release.yml`): пуш тега `v*` собирает
  `Adalight.exe` (PyInstaller) на Windows-раннере и публикует GitHub Release.
  Тег должен совпадать с версией в `pyproject.toml`.
